Oscar Guidelines: Scraper + Criteria Tree Explorer

A full-stack pipeline that turns Oscar Health medical guideline PDFs into browsable decision trees.

What it does: Discovers PDFs from Oscar's clinical guidelines page, downloads them, extracts text, and uses an LLM (Groq / Llama 3.3 70B) to turn "initial" medical necessity criteria into structured JSON decision trees. A React UI lets you browse policies and expand/collapse AND/OR criteria.

Code: github.com/jasroopdhingra/oscar

How it was built

Backend first: FastAPI app, SQLAlchemy models (policies, downloads, structured_policies), and SQLite. Then the pipeline services in order — scraper to discover PDF links from the guidelines page, downloader to fetch and store files with retries and throttling, extractor (pdfplumber + initial-section regex), and structurer (Groq, JSON mode, Pydantic validation). A CLI (run_pipeline.py) runs discover → download → structure in one go or step-by-step; API routes expose the same triggers plus GETs for policy list and detail.

Frontend next: React + Vite + TypeScript, Tailwind. Policies list with filter tabs (All / Structured / Unstructured), then policy detail page and a recursive criteria tree component with AND/OR badges and expand/collapse. API client talks to the backend; proxy in Vite for local dev. After that, full pipeline run to populate the DB and structure 10+ guidelines, then README and docs updated with setup, run instructions, and the initial-only logic.

Pipeline

  1. Discover — Scrape the guidelines page; parse Next.js __NEXT_DATA__ to get PDF links and metadata. Upsert into SQLite by pdf_url.
  2. Download — Resolve each listing URL to the real PDF (Contentful CDN) via the policy page's __NEXT_DATA__. Stream to disk with retries and throttling.
  3. Structure — Extract text with pdfplumber (first 15 pages / 30k chars). Filter to "initial" criteria only (regex), then send to the LLM with a strict schema. Validate with Pydantic; store JSON or validation errors.

Initial-only logic

Guidelines often separate "Initial Authorization" from "Continuation/Renewal" criteria. Two layers:

  • Layer 1 — In the extractor: detect initial vs continuation section headers and slice the text so only the initial block is sent to the LLM.
  • Layer 2 — In the system prompt: instruct the model to output only initial criteria and to ignore continuation, exclusions, and appendices.

If no such headers exist, the full extracted text is sent and the prompt alone enforces the scope.

Stack

  • Backend: FastAPI, SQLAlchemy (SQLite), Pydantic. Services: scraper, downloader, extractor, structurer (Groq, JSON mode).
  • Frontend: React, Vite, TypeScript, Tailwind. Policies list (filter by structured/unstructured), policy detail, recursive criteria tree with AND/OR badges and expand/collapse.
  • Run: python run_pipeline.py for discover → download → structure; or POST to /api/pipeline/*. UI: backend on 8000, frontend on 5173.

Decisions

  • __NEXT_DATA__ over DOM scraping — stable, structured access to links and content.
  • Two-step PDF URL resolution — listing links are Next routes; the actual file URL lives in the policy page's JSON.
  • Pydantic validation on every LLM response — invalid output is stored in validation_error instead of overwriting.
  • Idempotent steps — discover by unique URL; download and structure skip already-succeeded rows.
